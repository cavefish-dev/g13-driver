# v0.3 — macro sequences & shell-command bindings

- **Status:** open
- **Target:** v0.3
- **Updated:** 2026-06-30

## Goal
Extend bindings beyond single shortcuts: typed macro sequences and launching commands.

## Tasks
- [ ] Extend config to the future-compatible table form (non-breaking — plain string
      bindings must keep working):
      `[keys.G2] type = "macro" sequence = [...]`,
      `[keys.G3] type = "command" run = "notepad.exe"`.
- [ ] Macro executor: type strings + named keys in order, with optional delays.
- [ ] Command executor: spawn a process detached; log failures, never block dispatch.
- [ ] Tests for config parsing of both forms and macro expansion.

## Acceptance
A macro key types a multi-key sequence into a focused app; a command key launches a process.
Old string-only configs still load unchanged.

## Notes
- See "Future-compatible extended form" in `docs/design-spec.md`.
