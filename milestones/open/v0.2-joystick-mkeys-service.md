# v0.2 — joystick, M-key profiles, Windows Service

- **Status:** open
- **Target:** v0.2
- **Updated:** 2026-06-30

## Goal
Make the driver usable for gaming and run unattended: joystick input, profile switching
via the M-keys, and installation as a background Windows Service.

## Tasks
- [ ] Decode joystick X/Y (report bytes 5–6) and the joystick button (byte 4).
- [ ] Map joystick -> WASD and/or mouse movement (config-driven).
- [ ] Decode M-keys (M1/M2/M3/MR) from byte 4.
- [ ] Profile switching: M-keys select among multiple `[keys]` sets in config.
- [ ] Run as a Windows Service (install/uninstall), promoted from console app.

## Acceptance
Joystick drives movement in a test app; M-keys swap active profile live; service installs,
auto-starts, and survives logout.

## Notes
- Keep platform isolation: service + raw input stay behind `#[cfg(windows)]`.
- Joystick is a hot path — consider deadzone + polling cadence.
