## Hardware smoke test (manual — no unit tests on the USB transfer) — PASSED 2026-07-16
- [x] Backlight lights at the default color on startup.
- [x] Switching M1/M2/M3 (GUI Profiles tab) changes the color per slot.
- [x] A slot set to "use default" shows the global default color.
- [x] Brightness slider dims the pad; 0% turns it off.
- [x] The active profile's M-key LED lights (and only that one); toggling the
      Settings checkbox off turns the M-key LEDs off.
- [x] Unplug/replug the G13 — the correct color/LEDs re-apply automatically.
- [x] No G13 connected — app runs normally, no crash, warnings only.

## Notes from bring-up
- Colors, per-slot switching, brightness, and the M-key indicator all work; RGB
  channel order is correct (a `#FF0000` config shows red).
- **`#FFFFFF` renders warm/pinkish** on the G13 — the keypad's backlight LEDs are
  not color-balanced, so equal RGB is not neutral white. This is a hardware trait,
  not a driver bug.
- Gotcha for future manual testing: the app loads `config.toml` **next to the exe
  first** (`resolve_config_path`), so edit `target/release/config.toml` (or use the
  GUI) — editing the repo-root `config.toml` has no effect on the release build.
