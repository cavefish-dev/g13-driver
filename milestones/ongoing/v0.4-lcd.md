# v0.4 — LCD display

- **Status:** ongoing
- **Target:** v0.4
- **Updated:** 2026-07-16

## Goal
Drive the G13's 160x43 monochrome LCD: show active profile and basic system info.

## Tasks
- [x] Reverse-engineer / implement the LCD framebuffer write (USB control/bulk transfer).
- [x] 1-bit framebuffer abstraction (160x43) + simple text rendering.
- [x] Render active profile name; optional CPU/clock readout.

## Acceptance
Text appears on the physical LCD and updates on profile switch.

## Hardware smoke test (manual — no unit tests on the USB transfer)
- [ ] Text renders on the physical LCD (init transfer + endpoint 0x02 confirmed).
- [ ] Title + mode box show; mode reflects Active vs Dry-run.
- [ ] Profile line updates when switching M1/M2/M3.
- [ ] Last-action line updates on a G-key / thumb-button press and persists.
- [ ] Long profile/combo names truncate without corrupting the display.
- [ ] Unplug/replug — the frame re-appears automatically.
- [ ] No G13 connected — app runs normally, no crash, warnings only.

## Notes
- If the LCD stays blank, the init transfer or endpoint type (interrupt vs bulk)
  is the suspect — the GUI preview confirms the framebuffer/render is correct,
  isolating the fault to the USB write.
- Gotcha: the release build loads `target/release/config.toml` (next to the exe),
  not the repo-root one.
